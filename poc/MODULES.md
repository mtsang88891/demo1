## Create-Once Secret Pattern (null_resource)

Use this pattern when a secret must be created in Key Vault once and never overwritten by Terraform,
even if the variable value changes. The az CLI check ensures idempotency.

```hcl
resource "null_resource" "kv_secret_once" {
  triggers = {
    kv_name     = var.key_vault_name
    secret_name = var.secret_name
  }

  provisioner "local-exec" {
    command = <<-EOT
      set -e
      KV="${var.key_vault_name}"
      SECRET="${var.secret_name}"
      VALUE="${var.secret_value}"
      if az keyvault secret show --vault-name "$KV" --name "$SECRET" >/dev/null 2>&1; then
        echo "Secret $SECRET already exists in $KV - skipping."
      else
        az keyvault secret set --vault-name "$KV" --name "$SECRET" --value "$VALUE" >/dev/null
        echo "Created secret $SECRET in $KV."
      fi
    EOT
    interpreter = ["/bin/bash", "-c"]
  }
}
```
